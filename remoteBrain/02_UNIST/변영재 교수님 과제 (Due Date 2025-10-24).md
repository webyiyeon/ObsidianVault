### **산업 현장의 안전·보건·환경(SHE) 통합 관리를 위한 웹 기반 시스템 구축**

#### 제1장 서론

##### 1.1 연구 배경 및 필요성

현대 산업사회에서 기업의 지속가능성은 단순한 재무적 성과를 넘어, 안전(Safety)·보건(Health)·환경(Environment) 관리 수준에 의해 종합적으로 평가되고 있다. 특히 화학 산업은 생산 공정 전반에서 안전사고, 유해물질 노출, 환경오염 등 다양한 위험요소가 존재하기 때문에, 체계적이고 통합적인 안전·보건·환경(SHE; Safety, Health, Environment) 경영이 필수적이다.

SK Chemicals는 “인간과 환경을 핵심 가치로 인식하고, 이해관계자의 행복을 추구하기 위해 안전·보건·환경(SHE)를 최우선 가치로 삼는다”는 경영 원칙을 기반으로, SHE 체계를 전사적으로 운영하고 있다. 최신 개발 언어와 환경을 충분히 반영하지 못하고 있다. 또한 결재 흐름이 단일 구조로 고정되어 있어, 부서별 승인 절차나 병렬 결재와 같은 실무 형태를 반영하기 어렵다. 일부 기능은 단순 데이터 입력 수준에 머물러 활용도가 낮으며, 서버 또한 온프레미스 환경에 한정되어 있어 유지보수와 확장성 측면에서 비효율이 존재하였다. 이에 따라 본 연구에서는 시스템의 기능 고도화와 결재 체계 개선, 그리고 클라우드 기반 서버 환경으로의 전환을 목표로 하였다.또한 SHE 관련 지표나 모니터링 결과가 통합적으로 시각화되지 않아, 실시간 의사결정 지원 측면에서도 한계가 나타난다. 이러한 문제는 데이터의 누적 및 관리 자동화가 필요한 현 산업 환경에서, 보다 효율적이고 통합적인 SHE 데이터 관리·시각화 체계의 구축 필요성을 시사한다.


##### 1.2 연구 목적 및 구현 방향

본 연구는 SK Chemicals의 기존 SHE 경영 체계를 분석하고, 그 한계를 개선하기 위한 기술적 접근으로 SHE 관련 데이터를 효율적으로 관리하고 시각화할 수 있는 웹 기반 플랫폼을 설계·구현하는 것을 목적으로 한다. 해당 플랫폼은 현재 재직 중인 회사의 내부 프레임워크를 기반으로 개발되며, 데이터의 표준화, 접근성 향상, 운영 효율화를 주요 개선 방향으로 설정한다.

구체적으로는, 기존 시스템에서 분산되어 있던 데이터를 통합 관리할 수 있는 데이터 관리 모듈을 구축하고, 각종 SHE 지표를 직관적으로 파악할 수 있는 대시보드 형태의 시각화 기능을 구현한다. 또한 사용자 접근 권한, 데이터 추적, 자동 보고 기능 등을 포함한 구조적 설계를 통해, 기업 내부 SHE 관리의 신뢰성과 효율성을 동시에 향상시키는 것을 목표로 한다.

1. 산업 현장의 SHE 데이터를 통합 관리할 수 있는 웹 기반 플랫폼 구축
2. 자동화된 빌드·배포 파이프라인(Jenkins)을 통한 안정적인 운영 환경 조성
3. 데이터 중심의 위험 관리 체계 구현
4. 관리자와 현장 실무진 모두가 활용 가능한 사용자 친화적 UI/UX 제공 

본 연구를 통해 제안하는 플랫폼은 기술적 개선뿐 아니라, 기업의 지속가능경영 실현과 SHE 데이터 거버넌스 강화에도 기여할 수 있을 것으로 기대된다.


##### 1.3 이론적 배경

본 연구는 안전·보건·환경(SHE)의 통합 관리모델 구축에 관한 연구(김성빈·홍재훈·류병태 외 3인, 2006)를 비롯한 관련 선행연구들을 토대로 이론적 배경을 마련하였다. 기존 연구들은 산업현장에서 안전(Safety), 보건(Health), 환경(Environment)을 각각 분리하여 관리하던 체계의 비효율성을 지적하고, 이를 통합적으로 관리하기 위한 다양한 모델을 제시하였다.

그중 KW_SHE 모델은 위험요소의 식별–평가–개선–사후관리로 이어지는 순환 구조를 제안하며, PDCA(Plan–Do–Check–Act) 사이클을 기반으로 한 지속적 개선체계를 강조하였다. 또한 이후 연구들에서는 이 이론적 모델을 실제 산업현장에 적용하기 위한 구체적 접근이 이루어졌다. 예를 들어, 공정안전관리체계를 기반으로 한 통합안전관리시스템(S/W) 개발(김정원·고재욱, 2013)에서는 공정안전보고서 자동 생성 기능과 통합 메뉴 체계를 통해 정보의 효율적 관리 방안을 제시하였으며, 화학공장에서의 안전·보건·환경 통합체제 구축 연구(임차순·김성빈·김기수 외 3인, 2010)에서는 관리 문서의 표준화를 통한 통합체계 확립을 제안하였다. 또한 가스공정의 웹기반 통합 안전, 보건, 환경 시스템 WISHE(Web-based Integrated Safety, Health and Environment System) 연구(한경훈·박정수·정기택 외 3인, 2014)에서는 웹 환경에서 안전점검, 사고이력, 환경데이터를 통합 관리하는 시스템 아키텍처를 제시하며 SHE 관리의 정보화·자동화 방향성을 구체화하였다.

따라서 본 연구는 이러한 선행연구들의 관리적 개념을 확장하여, KW_SHE 모델의 핵심 이념을 실제 웹 정보시스템(Web Application) 형태로 구현함으로써 이론적 모델이 현실 산업 환경에서 활용 가능한 형태로 발전할 수 있음을 검증하고자 한다.



#### 제 2장. 본론

##### 2.1 전체 시스템 구조

본 시스템은 3-tier 구조(3계층 아키텍처)로 설계되었으며, 이는 사용자 인터페이스, 애플리케이션 로직, 데이터 관리 계층을 분리함으로써 유지보수성, 확장성, 안정성을 확보하기 위한 설계 방식이다.

- WAS 서버
	- Frontend: React.js를 사용하여 사용자 인터페이스(UI)와 데이터 시각화 대시보드 구현
	- Backend: Spring Boot 기반으로 RESTful API 설계 및 비즈니스 로직 처리
- Database: MSSQL을 사용하여 안전점검, 개선조치, 환경 측정, 보건 등의 데이터를 구조화 저장
- Build/Deploy: Jenkins를 통한 CI/CD 파이프라인 구축으로 코드 변경 시 자동 빌드·배포 수행


##### 2.1.1 WAS 서버 구조

WAS(Web Application Server)는 본 시스템의 핵심이며, Frontend, Batch, Backend, Deploy 환경을 포함한다. 각 계층은 다음과 같이 구성된다.

- Frontend (사용자 인터페이스)
    - React.js 기반으로 구현되어 있으며, SHE 담당자 및 관리자들이 현장 데이터를 조회하고 조치 상태를 관리할 수 있도록 구성된다.
    - 컴포넌트 단위 설계를 통해 각 기능(안전점검, 작업허가, 안전환경실시계획, 가동전안전점검, 변경관리 등)을 모듈화함으로써 코드의 재사용성을 높였다.
    - Axios를 이용해 서버 데이터의 캐싱과 비동기 요청 상태를 효율적으로 관리하며, 데이터 변경 시 자동으로 UI가 갱신되어 실시간성이 확보된다.
    - Wijmo의 React Chart 컴포넌트를 활용하여 주요 지표를 시각화하고, 대시보드 형태로 데이터를 한눈에 확인할 수 있도록 구현하였다.
    - 사용자 권한에 따라 접근 가능한 메뉴 구성을 다르게 하여 보안성을 강화했다.

- Backend (비즈니스 로직 처리)
    - Spring Boot 프레임워크를 기반으로 하며, RESTful API를 통해 프런트엔드와 데이터를 교환한다.
    - 주요 기능은 다음과 같다:
        1. 위험요소 등록/조회/수정/삭제 API
        2. 개선조치 승인 및 완료 처리 로직
        3. 환경·보건 데이터의 주기적 수집 및 통계화
        4. JWT 기반 인증(Authentication) 및 접근 제어(Authorization)
    - 서비스 레이어(Service Layer)를 중심으로 비즈니스 로직을 분리하여 유지보수가 용이하도록 하였으며, MyBatis를 이용해 SQL 쿼리를 관리하고 DB와 연동한다.
    - 또한, 에러 로그 및 시스템 로그를 파일과 DB 양쪽에 기록하여 장애 발생 시 신속한 원인 추적이 가능하다.


##### 2.1.2 Database (데이터 관리 계층)

서비스 레이어(Service Layer)를 중심으로 비즈니스 로직을 분리하여 유지보수가 용이하도록 하였으며, MyBatis를 이용해 SQL 쿼리를 관리하고 DB와 연동한다.안전·보건·환경 데이터를 하나의 DB에서 관리하되, 뷰(View) 또는 스토어드 프로시저(Stored Procedure)를 통해 부서별 또는 업무 목적별 데이터 접근이 가능하도록 구조화했다.


##### 2.1.3 Build/Deploy (자동화 배포 환경)

개발 및 운영 효율성을 위해 Jenkins 기반 CI/CD 파이프라인을 구축하였다.

- 개발자가 BitBucket에 코드를 push 한 후, Jenkins에 트리거를 주면 Jenkins가 자동으로 빌드를 수행하고, 테스트 및 정적 분석(Gradle build)을 거쳐 WAS 서버에 배포된다. BitBucket을 바라본 다음 push 발생 시 자동 수행하는 방법도 있었으나, 현장 업무 특성 상 허가를 받아야 다음 단계로 진행되는 경우도 있기 때문에 업무시간이 아닌 점심시간 또는 퇴근 후에 진행하도록 설정하였다.
- Jenkinsfile 스크립트를 이용하여 빌드 → 테스트 → 배포의 전체 단계를 자동화했으며, 환경별(개발/운영)로 Profile을 분리하여 안전한 배포를 보장한다.
- AWS CodeDeploy를 통해 AWS Linux 2023인 WAS 서버에 build된 파일을 압축해 전송하고, 전송 후에는 appspec.yml에 작성된 순서에 따라 bash script를 실행하여 배포를 진행한다. `systemctl daemon-reload`, `service restart` 명령을 포함해 서비스 중단 시간을 최소화하였다. 또한, 배포 로그와 시스템 로그를 자동 백업하여 이력 추적이 가능하다.


##### 2.2 시스템 통신 및 보안 고려사항

- HTTPS 기반 통신을 적용하여 모든 데이터 전송 시 암호화를 보장한다.
- JWT 토큰을 이용해 사용자 인증을 처리하며, API 호출 시 헤더 검증 절차를 통해 비인가 접근을 차단한다.
- 로그 및 감시 시스템은 Linux의 `journalctl`과 `systemd` 서비스 로그를 활용하여 오류 발생 시 실시간 모니터링이 가능하다.


##### 2.3 설계의 기대 효과

이러한 3-tier 구조를 통해 시스템은 다음과 같은 장점을 가진다.

1. 유지보수성(Maintainability): 기능 단위의 독립적인 코드 관리가 가능함.
2. 안정성(Stability): CI/CD 자동화로 배포 오류 최소화 및 서비스 중단 위험 감소.
3. 데이터 일관성(Consistency): DB 통합 관리로 중복 및 누락 방지.
4. 보안성(Security): 암호화 통신 및 권한 기반 접근제어로 민감 정보 보호.



##### 2.4 주요 기능

본 시스템은 산업 현장에서 수행되는 안전, 보건, 환경(SHE) 관련 주요 업무를 효율적으로 관리하기 위해 구축되었다. 각 기능은 기본적으로 등록, 결재, 진행 상태 추적의 단계로 이루어져 있으며,  
모든 데이터가 시스템 내에서 일원화되어 관리된다.

주요 기능으로는 작업허가서, 가동 전 안전점검, 변경관리, 안전환경작업실시계획서, 정기 안전점검 등이 있다. 각 업무는 담당자가 관련 정보를 등록하면, 승인 절차를 거쳐 최종 결재된 후 진행 상황을 추적할 수 있도록 구성되어 있다. 예를 들어, 작업허가서는 현장 담당자가 작업 내용을 등록하면 관리자가 승인하고, 작업 완료 후 결과가 기록되어 전체 이력이 남는다. 가동 전 안전점검이나 변경관리 또한 유사한 절차로 진행되어, 각 단계별 상태가 명확히 구분되고 관리된다.

보건 영역에서는 건강검진 결과와 인바디 측정 데이터를 관리하며, 검사 결과의 등록과 검토 이력을 남겨 개인별 건강 상태를 추적할 수 있다. 환경 영역에서는 환경 운영일지 작성, 폐기물 처리 이력, 오염물질 배출 관리 등이 포함되며, 각 항목의 데이터가 등록·승인·확인 절차를 통해 체계적으로 관리된다.

모든 업무 프로세스는 권한에 따라 접근 범위가 구분되며, 등록된 정보는 대시보드에서 실시간으로 반영되어 조치 현황과 진행 상태를 시각적으로 확인할 수 있다. 이와 같은 구조를 통해 현장의 다양한 SHE 데이터를 체계적으로 관리하고, 업무의 흐름과 결과를 명확하게 추적할 수 있도록 하였다.



#### 제 3장. 결론 및 향후 계획

##### 3.1 결론 및 향후 계획
산업 현장에서의 안전, 보건, 환경 데이터는 부서별로 분리되어 관리되는 경우가 많아 중복 입력과 누락이 빈번하게 발생한다. 본 시스템은 이러한 문제를 해결하기 위해 데이터를 하나의 플랫폼에서 일원화하여 관리하도록 설계하였다. 이를 통해 각 부서 간 정보가 연계되고, 데이터 흐름의 일관성이 확보되었다. 또한 Jenkins를 이용한 자동 배포 환경을 구축함으로써 운영자가 수동으로 수행하던 빌드와 배포 과정을 자동화하여 업무 효율성을 높였다. React 기반의 사용자 인터페이스는 현장 담당자들이 별도의 기술적 지식 없이도 손쉽게 접근하고 사용할 수 있도록 직관적으로 구성하였다. 대시보드 화면에서는 점검 현황과 조치 진행 상태를 실시간으로 확인할 수 있어, 관리자가 빠르게 상황을 파악하고 의사결정을 내릴 수 있다. 점검 등록, 검토, 조치, 완료 확인의 일련의 프로세스를 시스템 내부 로직으로 구현하여 PDCA 순환 구조를 자연스럽게 반영하였으며, 이를 통해 지속적인 개선과 성과 추적이 가능하게 되었다.

본 연구는 산업 현장의 안전·보건·환경 데이터를 체계적으로 관리하기 위한 웹 기반 통합 관리 시스템 구축 방안을 제시하였다. 기존 문헌에서 제시된 KW_SHE 통합 관리 모델의 개념을 실제 정보시스템 형태로 구현함으로써, 관리 중심의 이론적 모델을 산업 현장에서 실제로 운영 가능한 수준으로 구체화하였다. 향후에는 시스템 내 데이터 분석 기능을 강화하고, 점검 결과를 자동 리포트 형태로 생성하는 등 현장 중심의 사용자 편의 기능을 점진적으로 확장해 나갈 계획이다.
